# ContactHub プロジェクト運用ルール

## Git ワークフロー

このプロジェクトは **main ブランチで直接作業する** 運用とする。

### やらないこと

- `git worktree add` で worktree を新規作成すること
- 機能ごとに新しいブランチを切ること（`claude/xxx` のような自動命名ブランチを含む）
- superpowers の `using-git-worktrees` スキルの実行
- `EnterWorktree` ツールの使用

### やること

- 変更は直接 `main` ブランチにコミットする
- コミット前に必ずビルド・型チェックを通す
- リモート (`origin/main`) との同期を意識する

### 例外

ユーザーが明示的に「ブランチを切って」「worktree で作業して」と指示した場合のみ、新規ブランチ／worktree を作成してよい。それ以外は確認すること。
